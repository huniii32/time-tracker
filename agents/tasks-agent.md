# Tasks Agent

## 역할

업무 보드와 업무 상태 관리를 담당한다.

## 담당 범위

- 업무 등록
- 업무 상태 관리
- 우선순위 관리
- 마감기한 관리
- 피드백 관리
- 만족도 기록

## 구현할 기능

- 업무 데이터 요구사항
- 상태값 기준
- 우선순위 기준
- 마감 임박 표시 기준
- 완료 업무 표시 기준

## 구현하지 않을 기능

- 실제 CRUD 코드 작성
- 칸반 드래그 앤 드롭
- 복잡한 차트
- 외부 캘린더 연동

## 참조해야 할 데이터

- 업무
- 상태
- 우선순위
- 마감기한
- 태그
- `user_id`

## UI 원칙

- 상태와 마감기한이 먼저 보이게 한다.
- 완료 업무는 명확히 구분한다.
- 모바일에서 필터 조작이 쉬워야 한다.

## 완료 기준

- 업무 보드의 필드와 흐름이 정리되어 있다.
- 업무 상태 전환 기준이 명확하다.

## 검증 기준

- 업무 CRUD
- 상태 필터
- 우선순위 필터
- 마감기한 정렬

## 주의사항

이번 문서 작업에서는 실제 업무 CRUD 코드를 작성하지 않는다.
