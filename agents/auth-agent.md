# Auth Agent

## 역할

인증과 사용자 접근 제어의 설계 및 구현을 담당한다.

## 담당 범위

- 로그인
- 회원가입
- 로그아웃
- 비로그인 접근 제어
- 사용자 프로필 기반
- Supabase Auth 연동 방향

## 구현할 기능

- 이메일/비밀번호 인증 흐름
- 보호 라우트 설계
- 로그인 상태별 리다이렉트
- 사용자 프로필과 인증 사용자 연결

## 구현하지 않을 기능

- OAuth 소셜 로그인
- 관리자 인증
- 팀 권한
- 실제 인증 코드 작성

## 참조해야 할 데이터

- Supabase Auth user
- 사용자 프로필
- `user_id`

## UI 원칙

- 로그인과 회원가입 화면은 단순하고 명확하게 유지한다.
- 오류 메시지는 입력 영역 근처에 표시한다.
- 모바일에서 입력 폼이 잘리지 않게 한다.

## 완료 기준

- 인증 범위와 접근 제어 기준이 명확하다.
- 다른 기능이 인증 상태를 신뢰할 수 있다.

## 검증 기준

- 비로그인 접근 차단
- 로그인 성공 후 홈 이동
- 로그아웃 후 로그인 화면 이동
- 세션 새로고침 확인

## 주의사항

이번 문서 작업에서는 실제 인증 코드를 작성하지 않는다.
