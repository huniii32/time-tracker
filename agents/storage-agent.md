# Storage Agent

## 역할

데이터 저장 원칙과 접근 계층을 담당한다.

## 담당 범위

- Supabase PostgreSQL 저장 원칙
- `user_id` 기반 데이터 접근
- RLS 적용 방향
- query layer 분리 원칙
- 타입 정의 원칙

## 구현할 기능

- 테이블 설계 방향 문서화
- 사용자 데이터 격리 원칙
- 쿼리 계층 분리 기준
- 타입 정의 위치 기준

## 구현하지 않을 기능

- 실제 Supabase 쿼리 코드 작성
- 실제 테이블 생성 SQL 작성
- 마이그레이션 실행
- DB 연결 구현

## 참조해야 할 데이터

- 사용자 프로필
- 노트
- 업무
- 미팅
- 시간 기록
- 위클리 피드백
- 목표
- 회고

## UI 원칙

이 에이전트는 UI를 직접 담당하지 않는다. 다만 UI가 저장소 구현 세부사항에 의존하지 않도록 데이터 접근 계층을 분리한다.

## 완료 기준

- 데이터 저장 원칙이 명확하다.
- 기능별 쿼리 파일 분리 기준이 정리되어 있다.

## 검증 기준

- 모든 주요 데이터에 `user_id` 포함
- RLS 정책 방향 확인
- 컴포넌트에서 DB 접근 세부사항이 분리되어 있는지 확인

## 주의사항

이번 문서 작업에서는 실제 Supabase 쿼리 코드를 작성하지 않는다.
