# PRD: 회사 적응기 / Onboarding Tracker

> 이 문서는 제품 요구사항 참고 문서다. 이 문서를 한 번에 모두 구현하지 않는다. 기능은 `MILESTONES.md` 기준으로 단계별 구현한다.

## 제품 개요

회사 적응기 또는 Onboarding Tracker는 개인용 회사 적응 기록 웹앱이다. 사용자는 로그인 후 회사 문화, 상사 관찰, 회사 용어, 동료 협업 방식, 업무, 회의, 감정, 학습, 시간 루틴, 주간 회고, 목표를 기록하고 관리한다.

## 사용자

- 개인 사용자 1인
- 회사 적응 과정을 체계적으로 기록하려는 신입 또는 전환 초기 구성원

## 기술 스택

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Supabase Auth
- Supabase PostgreSQL
- Vercel

## 화면 구조

하단 탭은 5개로 구성한다.

1. 홈
2. 노트
3. 업무
4. 루틴
5. 회고

홈:
- 입사 D+일차
- 오늘의 체크리스트
- 오늘의 업무
- 최근 기록

- 학습 시간 요약
- 감정 상태 요약

노트:
- 회사노트
- 상사노트
- 회사 단어 사전
- 동료노트
- 감정노트
- 학습노트
- 검색, 태그, 날짜 정렬

업무 및 미팅:
- 업무 보드
- 미팅 보드
- 상태 관리
- 우선순위 관리
- 마감기한 관리
- 결정사항과 액션아이템 추적

루틴:
- 데일리 시간 기록

- 목표 진행률 계산

회고:

- 감정 관리 포인트

## 인증 요구사항

- 이메일/비밀번호 로그인
- 회원가입
- 로그아웃
- 비로그인 접근 제어
- 사용자 프로필 관리

## 데이터 저장 원칙

- 모든 데이터는 Supabase PostgreSQL에 저장한다.
- 모든 주요 데이터는 `id`, `user_id`, `created_at`, `updated_at`을 가진다.
- 모든 데이터 접근은 현재 로그인한 사용자의 `user_id` 기준으로 제한한다.
- Supabase RLS로 사용자 데이터 격리를 보장한다.
- 데이터 접근 로직은 UI 컴포넌트와 분리한다.
