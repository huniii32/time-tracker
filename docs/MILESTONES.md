# MILESTONES.md

# 회사 적응기 기능 구현 순서

## 원칙

전체 기능을 한 번에 구현하지 않는다.

각 단계는 하나의 마일스톤으로만 진행한다.

이전 마일스톤이 검증을 통과해야 다음 마일스톤으로 넘어간다.

## Milestone 0: 운영 문서와 에이전트 구조

### 목표
Codex가 전체 기능을 한 번에 구현하지 않도록 개발 운영 구조를 만든다.

### 포함 범위
- AGENTS.md
- docs/PRD.md
- docs/MILESTONES.md
- docs/HARNESS.md
- 기능별 agents 문서
- 기본 폴더 구조

### 제외 범위
- 실제 앱 기능 구현
- 로그인 구현
- CRUD 구현
- UI 화면 구현

### 완료 기준
- 문서 구조가 생성되어 있다.
- 기능별 에이전트 파일이 생성되어 있다.
- HARNESS 기준이 정의되어 있다.

## Milestone 1: 인증 기반 구축

### 목표
로그인 기반 웹앱의 기본 구조를 만든다.

### 포함 범위
- Supabase 연결 준비
- 로그인
- 회원가입
- 로그아웃
- 비로그인 접근 제어
- AppShell
- BottomNav
- 홈 skeleton

### 제외 범위
- 노트 CRUD
- 업무 CRUD
- 미팅 CRUD
- 루틴 CRUD
- 회고 CRUD
- 차트
- AI 기능

### 완료 기준
- 회원가입이 가능하다.
- 로그인이 가능하다.
- 로그아웃이 가능하다.
- 로그인하지 않으면 홈에 접근할 수 없다.
- 로그인하면 홈 skeleton이 보인다.
- 하단 탭 네비게이션이 보인다.
- 빌드가 통과한다.
- 타입 에러가 없다.

## Milestone 2: 데이터 모델과 Query Layer

### 목표
기능 구현 전에 사용자별 데이터 접근 구조를 안정화한다.

### 포함 범위
- profiles 데이터 구조
- notes 데이터 구조
- tasks 데이터 구조
- meetings 데이터 구조
- time_logs 데이터 구조
- weekly_reviews 데이터 구조
- goals 데이터 구조
- reflections 데이터 구조
- 기능별 query layer 구조

### 제외 범위
- 전체 화면 UI 구현
- 복잡한 대시보드 계산
- 차트
- AI 기능

### 완료 기준
- 주요 데이터 타입이 정의되어 있다.
- 기능별 query layer 파일이 준비되어 있다.
- 모든 데이터 구조가 user_id 기준 접근을 고려한다.

## Milestone 3: 홈 대시보드 1차

### 목표
앱을 열었을 때 오늘의 상태를 볼 수 있게 한다.

### 포함 범위
- 입사 D+일차
- 오늘의 체크리스트
- 오늘의 업무 요약
- 최근 기록
- 이번 주 목표 진행률 placeholder
- 감정 상태 placeholder

### 완료 기준
- 홈에서 핵심 요약 카드가 보인다.
- 빈 상태 UI가 자연스럽게 보인다.
- 모바일 화면에서 깨지지 않는다.

## Milestone 4: 노트 기능

### 목표
회사 적응 기록의 핵심 기능을 구현한다.

### 포함 범위
- 회사노트
- 상사노트
- 회사 단어 사전
- 동료노트
- 감정노트
- 학습노트
- 노트 생성
- 노트 목록
- 노트 상세
- 노트 수정
- 노트 삭제
- 검색
- 태그

### 완료 기준
- 노트를 생성, 조회, 수정, 삭제할 수 있다.
- 노트 타입별 입력 폼이 구분된다.
- 현재 로그인한 사용자 데이터만 보인다.

## Milestone 5: 업무 보드

### 목표
실제 회사 업무를 관리할 수 있게 한다.

### 포함 범위
- 업무 목록
- 업무 작성
- 업무 상세
- 업무 수정
- 업무 삭제
- 상태 변경
- 우선순위 표시
- 마감 임박 표시
- 완료 처리

### 완료 기준
- 업무를 생성, 조회, 수정, 삭제할 수 있다.
- 상태별 필터링이 가능하다.
- 마감기한과 우선순위가 보인다.

## Milestone 6: 미팅 보드

### 목표
회의 내용과 액션아이템을 관리한다.

### 포함 범위
- 미팅 목록
- 미팅 작성
- 미팅 상세
- 미팅 수정
- 미팅 삭제
- 결정사항 기록
- 액션아이템 기록
- 반영 여부 체크

### 완료 기준
- 미팅을 생성, 조회, 수정, 삭제할 수 있다.
- 액션아이템과 반영 여부를 확인할 수 있다.

## Milestone 7: 루틴 기능

### 목표
퇴근 후 학습과 시간관리를 기록한다.

### 포함 범위
- 데일리 시간 기록
- 위클리 피드백
- 목표 통합
- 시간 계산
- 목표 진행률 계산

### 완료 기준
- 시간 기록을 남길 수 있다.
- 주간 피드백을 작성할 수 있다.
- 목표를 레벨별로 관리할 수 있다.

## Milestone 8: 회고 기능

### 목표
하루의 배움과 다음 행동을 정리한다.

### 포함 범위
- 회고 작성
- 회고 목록
- 회고 상세
- 회고 수정
- 회고 삭제
- 최근 회고 요약

### 완료 기준
- 회고를 생성, 조회, 수정, 삭제할 수 있다.
- 최근 회고가 홈에 요약될 수 있다.

## Milestone 9: 홈 대시보드 고도화

### 목표
각 기능에서 쌓인 데이터를 홈에서 요약한다.

### 포함 범위
- 오늘 업무 수
- 마감 임박 업무
- 최근 노트
- 최근 회고
- 이번 주 코딩 공부 시간
- 이번 주 논문 리뷰 시간
- 감정 흐름
- 목표 진행률

### 완료 기준
- 홈에서 주요 기록이 요약된다.
- 실제 데이터 기반으로 계산된다.

## Milestone 10: 모바일 UX 및 배포

### 목표
iPhone과 PC에서 안정적으로 사용할 수 있게 한다.

### 포함 범위
- Vercel 배포
- iPhone Safari 확인
- 반응형 레이아웃 개선
- 입력 폼 터치 영역 개선
- 빈 상태 UI 개선
- 로딩 상태
- 에러 메시지 정리

### 완료 기준
- Vercel에 배포된다.
- 모바일 Safari에서 사용 가능하다.
- 주요 화면이 깨지지 않는다.
